# 第 8 章示例：LangGraph 工作流

这一章的示例用一个最小版 `AI 学习助手` 演示“状态 + 节点 + 条件分支”的工作流思路。

## 示例层级

- 层级：`教学版`
- 重点：理解状态、节点和条件边如何把复杂流程组织成图。
- 边界：示例先讲工作流思想，不依赖真实 LangGraph 运行时，也不覆盖持久化、人工介入 UI 或线上恢复机制。

## 你会学到什么

- 为什么复杂流程更适合图，而不是一条固定链
- 状态如何在多个节点之间传递
- 条件分支如何决定下一步走向

## 运行前提

- Python 3.10 或更高版本
- 不需要额外的第三方依赖

## 安装方式

这个示例不依赖外部包，直接运行即可。

## 运行命令

```bash
python3 main.py
```

## 预期输出

脚本会展示：

1. 问题分类
2. 路由到不同节点
3. 检索、澄清、总结之间的状态流转
4. 最终输出如何形成

## 学习重点

这个示例不追求真实 LangGraph API，而是先把工作流思想讲清楚。

你应该重点观察：

- 每个节点只做一件事
- 状态对象是整个流程的共享载体
- 条件边决定下一步走向
- 失败时要有兜底路径
