# 第 6 章示例：RAG 进阶优化

这一章的示例演示 `AI 学习助手` 的 RAG 优化思路。

## 示例层级

- 层级：`教学版`
- 重点：比较不同切分、检索和上下文组织策略的效果差异。
- 边界：示例保留优化思路，但仍是可控的课程样例，不等于真实生产级检索评测或线上召回系统。

你会学到：

- 为什么基础 RAG 常常“能跑但不好用”
- 如何比较不同切分粒度
- 如何做更严格的检索与重排
- 如何控制上下文长度与回答边界

## 运行前提

- Python 3.10+
- 不需要 API key

## 安装

```bash
pip install -r requirements.txt
```

## 运行

```bash
python3 main.py
```

示例会从 `../../shared/datasets/chapter-06/course_materials.json` 读取课程资料。

## 预期输出

脚本会对比两种 RAG 策略：

- baseline：简单切分 + 直接取前几个片段
- optimized：更合理的切分 + 重排 + 上下文压缩

你会看到每种策略检索到的片段和最终答案的差异。

## 你在学什么

这个示例不只是回答问题，而是教你如何定位 RAG 的问题到底出在：

1. 切分
2. 检索
3. 上下文组织
4. 生成约束

这是 `AI 学习助手` 走向可控、可调试 RAG 的关键一步。

## 为什么把数据拆出去

把课程资料放到 `shared/datasets`，可以把“要检索什么”与“如何检索”分开处理：

- 调整教学语料时，不会把优化实验逻辑一起改乱
- 后续更容易替换成新的资料集，复用同一套实验脚本
- 更接近真实工程里单独维护知识库、索引和应用逻辑的结构
