# 第 3 章示例：第一个链式应用

这个示例对应课程第 3 章，目标是把 `PromptTemplate`、一个最小的“模型”函数和输出解析串成一条完整链路，并补一版更接近真实 LCEL 接口的对照实现。

## 示例层级

- 层级：`教学版`
- 重点：理解链式应用如何把输入、编排和输出拆开。
- 边界：示例保留 LangChain 结构，但模型仍是本地模拟函数，不包含真实 API 调用、缓存或失败恢复。
- 对照：脚本里额外提供 `真实框架版`，改用 `ChatPromptTemplate + JsonOutputParser` 展示更接近工程代码的链式接口。

## 你将学到什么

- 如何把一次 AI 调用拆成输入、编排、输出三层
- 为什么链式结构比直接写一个大函数更容易维护
- 如何先做出一个可跑通的最小版本，再为后续 RAG 和 Agent 预留接口

## 运行前提

- Python 3.10+
- 只需要 `langchain-core`
- 不需要 API Key，也不需要额外服务

## 安装

在当前目录执行：

```bash
pip install -r requirements.txt
```

## 运行

```bash
python3 main.py
```

## 预期输出

程序会依次打印：

- `教学版`：纯文本学习计划
- `真实框架版`：结构化学习计划

## 你在学什么

这不是在学“一个聊天脚本”，而是在学一条可维护的 AI 流程：

`输入 -> PromptTemplate -> Runnable 模型 -> 输出解析 -> 结果`

示例代码先用本地模拟模型函数帮你理解链式结构，再用更接近真实框架接口的版本展示如何把结果直接做成结构化输出，方便下一步接入页面或服务层。
