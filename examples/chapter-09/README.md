# 第 9 章示例：评估、观测与调试

这个示例对应第 9 章，目标是给 `AI 学习助手` 建立一个最小可用的评估、观测和调试流程。

- 定义一个很小但可重复执行的评估集
- 跑一次可预测的 `AI 学习助手` 演示流程
- 为每次请求记录 trace 数据
- 根据预期要点做打分和缺失分析

## 你会学到什么

- 如何把“感觉还行”的测试方式变成可重复执行的检查
- 如何记录输入、路由、输出和耗时这些最小 trace
- 如何从回答里找出缺失要点，而不是只看有没有生成结果

## 运行前提

- Python 3.10 或更高版本
- 不需要第三方依赖

## 运行命令

```bash
cd /home/litianwei/langchain-learning-path/examples/chapter-09
python3 main.py
```

如果你只想检查单个用例，可以运行：

```bash
python3 main.py --case basic-01
```

## 预期输出

脚本会输出：

- 评估用例列表
- 每个问题对应的路由
- 分数和缺失要点摘要
- 一段适合调试的紧凑 trace

## 你在学什么

这一章不是为了让模型“看起来更聪明”，而是为了让系统变得可衡量、可比较、可定位问题。

这个脚本提供的是一个本地离线版的工程习惯：先定义问题，再记录过程，再判断结果。以后即使你接入 LangSmith 或其他可观测平台，这套思路也不会变。
