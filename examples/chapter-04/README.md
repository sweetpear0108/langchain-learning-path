# 第 4 章示例：Prompt 工程与输出控制

这个示例对应课程第 4 章，目标是通过两个 Prompt 对比，直观看到“提示词怎么影响输出质量”和“为什么输出格式要被提前约束”。

## 你将学到什么

- 如何把一个模糊任务改写成可控任务
- 为什么输出格式必须在 Prompt 里先定义
- 如何让 `AI 学习助手` 输出更适合页面渲染的结构

## 运行前提

- Python 3.10+
- 只需要 `langchain-core`
- 不需要 API Key，也不需要额外服务

## 安装

在当前目录执行：

```bash
pip install -r requirements.txt
```

## 运行

```bash
python3 main.py
```

## 预期输出

程序会打印两段结果：

- `Loose Prompt`：提示词很松，输出更散
- `Controlled Prompt`：加入角色、约束和格式后，输出更稳定

## 你在学什么

这里不是在演示“更会说话”，而是在演示工程上的控制手段：

`任务层 -> 约束层 -> 格式层`

你会看到，相同的主题只要提示词结构不同，输出就会明显变化。
