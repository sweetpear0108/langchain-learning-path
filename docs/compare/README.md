# 方法对比页

这组页面不是为了背概念，而是为了回答一个更实际的问题:

- 我现在该选哪种方法开始做
- 什么时候应该升级方案
- 哪些场景不值得上更重的框架

如果你已经在跟着 `AI 学习助手` 这条主线学习，这些对比页的作用是帮你建立“选型视角”，而不是再重复一遍章节知识点。

## 建议阅读顺序

1. [Chain vs Agent](./chain-vs-agent.md)
2. [LangGraph vs 普通链式调用](./langgraph-vs-linear-chains.md)
3. [RAG vs Fine-tuning](./rag-vs-fine-tuning.md)
4. [LangChain vs LlamaIndex](./langchain-vs-llamaindex.md)

## 你能从这里得到什么

- 用一句话先判断某种方案是不是明显过度设计
- 看到每种方案的主要收益、代价和误判点
- 知道当前课程中的哪一章最适合补对应知识
- 知道什么时候该继续维持简单方案，什么时候该升级

## 和章节内容怎么配合

- 如果你还在第 3 章附近，优先看 [Chain vs Agent](./chain-vs-agent.md)
- 如果你正在做第 5、6 章，优先看 [RAG vs Fine-tuning](./rag-vs-fine-tuning.md)
- 如果你已经进到第 7、8 章，优先看 [LangGraph vs 普通链式调用](./langgraph-vs-linear-chains.md)
- 如果你开始关心整体框架边界，再看 [LangChain vs LlamaIndex](./langchain-vs-llamaindex.md)
