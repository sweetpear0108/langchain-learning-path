# FAQ

如果你现在不是“想知道为什么这样设计”，而是“命令跑不起来”，先看 [故障排查](./troubleshooting)。
如果你是读章节时被高频术语卡住，先看 [LangChain 术语表](./glossary)。

## 这套项目适合完全零基础吗

不适合完全没有编程基础的人。默认你已经会 Python 基础语法，知道什么是函数、字典、环境变量和命令行。

## 为什么不直接从 Agent 开始

因为 Agent 只是复杂应用的一部分。如果你没有先掌握 Prompt、链和 RAG，后面很容易只会调接口，不知道哪里出了问题。

## 为什么文档站放在 GitHub Pages，后端却不放

因为 GitHub Pages 适合静态前端，不适合安全地运行需要密钥的服务端逻辑。这个项目第一阶段先完成静态站，第二阶段再补独立后端。

## 每章都必须先看文档再跑代码吗

建议是。章节正文会说明“这一章为什么存在”，示例代码则让你真正把它跑起来。少掉任意一边，学习效果都会打折。

## 我需要一次性看完 10 章吗

不需要。更合理的方式是每章都跑一次示例，再进入下一章。
