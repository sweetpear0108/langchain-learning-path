# LangChain Learning Path 完整课程大纲

## 1. 课程总目标

完成本课程后，学习者应具备以下能力：

- 理解 LLM 应用开发的核心构成
- 掌握 LangChain 的主要抽象与基本用法
- 能独立实现基础链式应用
- 能构建基础 RAG 系统并做初步优化
- 能使用工具调用与 Agent 模式
- 能使用 LangGraph 组织多步骤工作流
- 能完成一个可部署、可演示、可继续扩展的综合项目

## 2. 课程组织方式

课程采用“章节 + 模块 + 练习 + 项目推进”的组织方式。

每章包含：

- 学习目标
- 前置知识
- 核心概念
- 最小示例
- 代码拆解
- 练习任务
- 常见问题
- 本章总结

## 3. 贯穿式项目主线

课程主项目：`AI 学习助手`

项目能力会逐章演进：

- 问答
- 结构化输出
- 多步骤链
- RAG 问答
- 检索优化
- Tool Calling
- Agent 化
- LangGraph 工作流化
- 可公开演示版本

## 4. 章节总览

1. LLM 应用基础
2. LangChain 核心抽象
3. 第一个链式应用
4. Prompt 工程与输出控制
5. RAG 入门

7. Tools 与 Agent
8. LangGraph 工作流

10. 综合实战与公开部署

## 5. 第一章：LLM 应用基础

### 章节目标

- 理解什么是 LLM 应用
- 区分模型、提示词、上下文、输出
- 了解聊天接口与补全接口差异
- 明确 LangChain 解决的是什么问题

### 模块划分

#### 1.1 什么是 LLM 应用

- 从“调用模型”到“构建应用”
- LLM 应用的基本组成
- 一个最小 AI 应用的输入输出过程

#### 1.2 核心概念入门

- Token
- Context Window
- Temperature
- Prompt
- Message
- Structured Output

#### 1.3 模型调用的基本方式

- 单轮问答
- 系统提示词
- 用户提示词
- 输出示例观察

#### 1.4 为什么需要框架

- 直接 SDK 调用的问题
- Prompt、模型、输出处理越来越复杂
- LangChain 的价值边界

### 本章实践

- 写一个最小问答脚本
- 观察不同提示词的输出差异
- 比较纯文本输出和 JSON 输出需求

### 本章产出

- `chapter-01` 示例代码
- 本章概念图

推荐站内给出三种学习模式：

- 速通模式：只看核心章节和最小示例
- 标准模式：按顺序完成全部章节
- 项目模式：围绕主项目一路升级

## 18. 章节间依赖关系

- 第 1 章是全课程入口
- 第 2 章是 LangChain 使用前提
- 第 3、4 章建立基础应用能力
- 第 5、6 章建立 RAG 能力
- 第 7、8 章进入复杂应用编排
- 第 9 章补工程化能力
- 第 10 章完成对外发布

## 19. 建议首发版本章节范围

如果只做首发 MVP，建议先上线以下内容：

1. 第一章：LLM 应用基础
2. 第二章：LangChain 核心抽象
3. 第三章：第一个链式应用
4. 第五章：RAG 入门

理由：

- 能快速构成完整认知闭环
- 难度梯度合理
- 读者能尽快看到实际效果

## 20. 结语

这套课程大纲的核心不是“尽可能多讲概念”，而是“让学习者沿着一条稳定主线逐步获得可交付能力”。

如果后续继续扩展，优先增加每章的示例代码、图解和练习，而不是先增加更多章节数量。
