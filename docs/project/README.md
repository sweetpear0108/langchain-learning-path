# 项目主线

这个仓库不是每章都换一个 Demo，而是围绕同一个主项目持续升级：`AI 学习助手`。

## 为什么要有主项目

如果每章都换一个不相关的小例子，学习者最后通常只会记住零散 API，不会形成完整工程能力。

主项目的作用是把每一章学到的能力都挂到同一条线上：

- 第 1 章：理解这个助手要解决什么问题
- 第 2 章：理解它的最小抽象结构
- 第 3 章：写出第一个链式版本
- 第 4 章：稳定输出格式
- 第 5 章：让它基于资料回答问题
- 第 6 章：优化检索与回答质量
- 第 7 章：让它能调用工具
- 第 8 章：把流程编排成图
- 第 9 章：为它增加评估、观测和调试能力
- 第 10 章：把它整理成可以公开展示的项目

## 主项目版本演进

### V1

输入一个学习主题，输出摘要和学习顺序建议。

### V2

输出格式固定，适合网页渲染和后续程序处理。

### V3

接入课程资料，具备基础 RAG 问答能力。

### V4

接入 Tool 与 Agent，能在不同任务间动态决策。

### V5

迁移到 LangGraph，用显式状态和流程图组织复杂任务。

### 发布版

文档站负责教学和导航，示例代码负责动手实践，第二阶段后端负责在线实验。
